Семантические проверки:

    В программе не должно быть одноимённых функций.
    У функции не должно быть одноимённых параметров.
    Поддерживаются только следующие встроенные типы: int, bool, списки и кортежи (длиной не меньше 2).
    В вызовах функций, определённых пользователем, типы формальных и фактических аргументов должны совпадать.
    Функция car принимает аргумент типа [T] и возвращает значение типа T.
    Функция cdr принимает аргумент типа [T] и возвращает значение типа [T].
    Функция null принимает аргумент типа [T] и возвращает значение типа bool.
    В условной операции типом условия может быть только bool, типы выражений в обеих ветвях должны совпадать.
    Не существует неявных приведений типа.
